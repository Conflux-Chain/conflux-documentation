# Conflux Network Development Guidelines

This file provides context and guidelines for LLMs assisting with Conflux Network development.

## Core Concepts
- **Conflux Network**: A high-performance Layer 1 consensus blockchain utilizing a unique Tree-Graph consensus algorithm.
- **Two Spaces**: Conflux operates with two distinct spaces to maximize compatibility and performance:
  - **Core Space**: The native Conflux environment. Uses `cfx:` addresses (e.g., `cfx:aaj...`). High throughput, sponsored transactions.
  - **eSpace**: Fully EVM-compatible environment. Uses standard `0x` hex addresses. Compatible with Ethereum tools (MetaMask, Hardhat, Foundry).
- **Consensus**: Tree-Graph (DAG) structure allowing concurrent block generation, combined with Proof of Work (PoW) and Proof of Stake (PoS) finality.

## Development Environments

### eSpace (EVM Compatible)
When developing for eSpace, treat it exactly like Ethereum or Binance Smart Chain.
- **RPC Endpoints**: 
  - Mainnet: `https://evm.confluxrpc.com`
  - Testnet: `https://evmtestnet.confluxrpc.com`
- **Chain IDs**: 
  - Mainnet: `1030`
  - Testnet: `71`
- **Tools**: Hardhat, Foundry, Truffle, Ethers.js, Viem, Web3.js.
- **Block Explorer**: `https://evm.confluxscan.net`

### Core Space (Native)
When developing for Core Space, use Conflux-specific tooling.
- **RPC Endpoints**:
  - Mainnet: `https://main.confluxrpc.com`
  - Testnet: `https://test.confluxrpc.com`
- **Network IDs**:
  - Mainnet: `1029`
  - Testnet: `1`
- **Tools**: `js-conflux-sdk`, `go-conflux-sdk`, `java-conflux-sdk`.
- **Address Format**: Base32 encoded with network prefix (`cfx:`, `cfxtest:`).
- **Block Explorer**: `https://confluxscan.net`

## Smart Contract Development
- **Solidity**: Fully supported in both spaces.
- **Cross-Space Call**: Conflux features a built-in internal contract (`CrossSpaceCall`) that allows contracts in Core Space to interact with contracts in eSpace and vice versa.
- **Sponsorship Mechanism**: Core Space supports transaction sponsorship, allowing users to interact with smart contracts without paying gas fees (the contract sponsor pays).

## Common Tasks & Code Snippets

### Connecting to eSpace (Ethers.js)
```javascript
const { ethers } = require("ethers");
const provider = new ethers.providers.JsonRpcProvider("https://evmtestnet.confluxrpc.com");
```

### Connecting to Core Space (js-conflux-sdk)
```javascript
const { Conflux } = require('js-conflux-sdk');
const conflux = new Conflux({
  url: 'https://test.confluxrpc.com',
  networkId: 1,
});
```

## Best Practices for LLMs
1. **Always clarify the space**: Before writing code, explicitly state whether the solution targets Core Space or eSpace.
2. **Address validation**: Ensure `cfx:` addresses are used for Core Space and `0x` addresses for eSpace. Never mix them directly without using the Cross-Space Call mapping.
3. **Gas estimation**: Conflux has a different gas pricing model than Ethereum. In Core Space, storage collateral is required.
4. **Tooling recommendations**: Default to recommending eSpace and standard EVM tools for new developers unless native Core Space features (like transaction sponsorship) are specifically requested.
